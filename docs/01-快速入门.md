# Mesh 快速入门

> 5 分钟上手 Mesh CLI（基于当前已实现功能）

---

## 1. 安装

### 1.1 本地编译

```bash
cd /Users/yuanze/Downloads/同步空间/mesh
go build -o mesh ./cmd/mesh/main.go
```

可选：放到 PATH

```bash
sudo mv ./mesh /usr/local/bin/mesh
```

### 1.2 验证

```bash
mesh --version
mesh --help
```

---

## 2. 初始化（首次必做）

```bash
mesh init --sync-space /Users/yuanze/Downloads/同步空间/.mesh-sync
```

执行后会：
- 创建配置文件：`~/.mesh/config.json`（可用 `MESH_CONFIG_PATH` 覆盖）
- 创建本地数据目录：`~/.mesh/`
- 写入本地数据库路径：`~/.mesh/mesh.db`（可用 `MESH_DB_PATH` 覆盖）

---

## 3. 日常使用

### 3.1 采集

```bash
mesh collect --source "claude" --content "决策：采用 Go 实现 mesh" --tag "技术,决策"
```

### 3.2 查询

```bash
mesh query "定价策略"
mesh query --source "claude" --tag "产品" --limit 5
```

### 3.3 列表

```bash
mesh list
mesh list --source "claude" --limit 10
```

---

## 4. Markdown 导入/导出

### 4.1 导入

```bash
mesh import /path/to/memory.md
mesh import --source "legacy-memory" /path/to/memory.md
```

支持格式示例：

```markdown
## 2026-03-10

### 决策：使用 Go 开发 Mesh
标签: 技术,决策
内容: 第一行
第二行
```

### 4.2 导出

```bash
mesh export
mesh export --output memory.md
```

---

## 5. 多端同步

```bash
mesh push
mesh pull
mesh sync
```

说明：
- `sync` 顺序是先 pull 再 push
- 同步目录使用锁文件 `.mesh.sync.lock` 避免并发
- 检测到潜在冲突时会先备份本地库：`mesh.db.bak.<timestamp>`

---

## 6. Agent 管理与上下文注入

### 6.1 注册/查看 Agent

```bash
mesh agent register --id "codex-mini2" --name "Codex Mini2" --type "codex" --device "mini2"
mesh agent list --limit 20
```

### 6.2 生成可注入上下文

```bash
mesh inject "定价策略"
mesh inject --source "claude" --tag "产品" --limit 8 --max-tokens 1200 --format markdown
```

---

## 7. 常见问题

### Q1: 提示“请先执行 mesh init”
先运行：

```bash
mesh init --sync-space <你的同步目录>
```

### Q2: 想切换数据库位置

```bash
export MESH_DB_PATH=/path/to/mesh.db
```

### Q3: 想切换配置位置

```bash
export MESH_CONFIG_PATH=/path/to/config.json
```
