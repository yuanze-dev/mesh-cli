# Mesh 多 AI 协作流程

> 极简的多 Agent 协作流程，实现人 + 多 Agent 的上下文共享

---

## 一、协作角色

### 1.1 角色定义

| 角色 | 人数 | 职责 |
|-----|------|------|
| **人类（晓力）** | 1 | 定义任务、跟进进展、最终验收 |
| **开发 Agent** | 多个 | 执行具体开发任务 |
| **Review Agent** | 1 | 代码审查、技术把关 |

### 1.2 角色标识

每个 AI/Agent 有唯一标识：

```
# 格式
{类型}_{序号}_{设备}

# 示例
claude_main_xiaolin     - xiaolin 电脑上的主控 Claude
claude_worker_1_xiaolin - xiaolin 电脑上的执行者 Claude 1
cursor_main_shawn       - shawn 电脑上的 Cursor
agent_backend_server    - 服务器上的后端 Agent
```

## 二、协作流程

### 2.1 完整流程图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    人类（晓力）定义任务                                  │
│                  查看 TODO.md → 分配任务给开发 Agent                      │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    开发 Agent 领取任务                                  │
│            读取宪法 → 读取协作流程 → 读取任务详情 → 开始工作              │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    开发 Agent 开始工作                                  │
│                   更新 TODO.md：状态改为 "进行中"                       │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    开发 Agent 完成任务                                  │
│         更新 TODO.md：状态改为 "待审查" → 提交代码审查                  │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    Review Agent 审查任务                                │
│            检查：代码质量、功能完整性、测试、文档                         │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    审查结果：通过 / 不通过                              │
│         通过：更新 TODO.md：状态改为 "完成" → 通知晓力                   │
│       不通过：返回修改意见 → 开发 Agent 修改后重新提交                    │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    人类（晓力）验收                                   │
│                    查看结果 → 确认任务完成 → 继续下一个任务               │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 极简流程（快速版）

对于简单任务，可以简化为：

```
晓力 → TODO.md(分配任务) → 开发 Agent → TODO.md(标记待审查) → Review Agent → TODO.md(标记完成) → 晓力验收
```

## 三、任务卡片格式

### 3.1 任务定义格式

```markdown
## [任务ID] 任务名称

**优先级**: P0/P1/P2/P3
**状态**: 待开始 / 进行中 / 完成 / 阻塞
**分配给**: [Agent ID]
**创建时间**: YYYY-MM-DD HH:MM

**任务描述**:
[清晰、具体、可验收的描述]

**验收标准**:
- [ ] 标准1
- [ ] 标准2
- [ ] 标准3

**依赖任务**: [任务ID]

**参考文档**:
- [文档路径]

**备注**:
[其他说明]
```

### 3.2 任务状态更新

**开发 Agent 完成**：
更新 `TODO.md` 中任务状态为：⏳ 待审查

**Review Agent 审查通过**：
更新 `TODO.md` 中任务状态为：✅ 完成

**状态说明**：
| 状态 | 说明 | 谁更新 |
|-----|------|--------|
| ⏳ 待开始 | 任务还未开始 | 人类 |
| 🔄 进行中 | 开发 Agent 正在开发 | 开发 Agent |
| ⏳ 待审查 | 开发完成，等待审查 | 开发 Agent |
| ✅ 完成 | 审查通过 | Review Agent |
| ❌ 阻塞 | 任务被阻塞，等待依赖完成 | - |

## 四、Agent 注册与识别

### 4.1 Agent 注册

```bash
# Agent 首次接入时注册
mesh agent register \
  --id "claude_worker_1_xiaolin" \
  --name "Claude Worker 1" \
  --type "claude" \
  --device "xiaolin" \
  --metadata "role=worker,version=4.6"
```

### 4.2 查看 Agent 列表

```bash
mesh agent list
```

输出示例：
```
ID                          Name              Type    Device    Last Seen
---------------------------------------------------------------------------
claude_main_xiaolin        Claude Main        claude  xiaolin   2026-03-13 18:00
claude_worker_1_xiaolin    Claude Worker 1    claude  xiaolin   2026-03-13 18:30
cursor_main_shawn          Cursor Main        cursor  shawn     2026-03-13 17:45
```

## 五、上下文注入

### 5.1 查询相关上下文

```bash
# Agent 开始任务前，查询相关上下文
mesh query "定价策略" --max-results 5 --format json
```

输出：
```json
{
  "insights": [
    {
      "id": "xxx",
      "source": "claude_main_xiaolin",
      "content": "...",
      "tags": ["定价", "产品"],
      "created_at": 1678900000
    },
    ...
  ]
}
```

### 5.2 注入到新对话

```bash
# 获取最近相关的认知，注入到新的 AI 对话
mesh inject --topic "定价策略" --max-tokens 2000 --format prompt
```

输出（可直接粘贴到新对话）：
```
# 上下文信息

## 相关认知（3条）

1. [claude_main_xiaolin, 2026-03-10]
   决策：定价策略采用分层定价，基础版免费，专业版 99 元/月

2. [cursor_main_shawn, 2026-03-11]
   竞品分析：主要竞品定价为 79-129 元/月

3. [agent_backend, 2026-03-12]
   技术实现：需要支持支付宝和微信支付
```

## 六、冲突处理

### 6.1 记录级冲突

当两个 Agent 同时修改同一条记录时：

1. **基于时间戳**：后写入的覆盖先写入的（简单策略）
2. **版本号**：支持更复杂的合并（未来）

### 6.2 任务级冲突

当两个 Agent 领取同一个任务时：

1. **先到先得**：第一个 `mesh collect --tag "任务,进行中"` 的获得任务
2. **状态检查**：领取前检查任务状态

## 七、示例：完整协作流程

### 场景：实现 `mesh collect` 命令

#### Step 1: 晓力查看 TODO.md

```bash
# 晓力打开 TODO.md，查看 TASK-001 状态
cat TODO.md
```

#### Step 2: 晓力分配任务给开发 Agent

```
晓力对开发 Agent 说："请执行 TASK-001：初始化 Go 项目"
```

#### Step 3: 开发 Agent 执行任务

```
开发 Agent：
1. 读取 CONSTITUTION.md
2. 读取 docs/04-多AI协作流程.md
3. 读取 tasks.md 中的 TASK-001
4. 开始开发
5. 更新 TODO.md：TASK-001 状态改为 ⏳ 待审查
```

#### Step 4: Review Agent 审查任务

```
Review Agent：
1. 查看 TODO.md 中 TASK-001 的状态
2. 检查代码质量、功能完整性
3. 运行测试
4. 更新 TODO.md：TASK-001 状态改为 ✅ 完成
```

#### Step 5: 晓力验收

```
晓力查看 TODO.md，确认 TASK-001 状态为 ✅ 完成，继续下一个任务
```

## 八、给 AI 的指令模板

### 模板1：开发 Agent 执行任务

```
你是 Mesh 项目的开发 Agent。

## 项目位置
../

## 你的任务
执行 TASK-001：初始化 Go 项目

## 必读文档（按顺序）
1. CONSTITUTION.md - 项目宪法
2. docs/04-多AI协作流程.md - 协作流程
3. tasks.md - 找到 TASK-001

## 执行流程
1. 读取上述文档
2. 开始执行任务
3. 完成后更新 TODO.md：状态改为 ⏳ 待审查

开始执行吧！
```

### 模板2：Review Agent 审查任务

```
你是 Mesh 项目的 Review Agent。

## 项目位置
../

## 你的任务
审查 TASK-001：初始化 Go 项目

## 验收检查项
- [ ] 代码质量（规范、注释、复杂度）
- [ ] 功能完整性（按需求实现）
- [ ] 测试覆盖（核心功能有测试）
- [ ] 文档更新（相关文档已更新）

## 执行流程
1. 查看 TODO.md 中任务状态
2. 检查代码质量
3. 运行测试
4. 审查通过后更新 TODO.md：状态改为 ✅ 完成

开始审查吧！
```
